# 让微调模型「识别整段课文朗读」· 数据与训练方案

> 背景：用 ~300 条「孤立短句」微调出的个性化模型，在**短句**上 CER ~16–18%，但拿去识别
> 一段 2 分钟的课文朗读（老舍《猫》）几乎是乱码（「猫的性格有些古怪」→「高的议者有些五百」）。
> 切句（VAD）也救不回来。本文档说明**为什么**、以及**怎么改**才能让长段朗读可用。
>
> 适用前提（已与产品确认）：**开放转写**（朗读文本事先未知）+ **可向小崔补录朗读数据**。

---

## 一、为什么现在不行（根因）

短句评测好、长段朗读崩，**不是因为「音频长」**，而是因为「课文朗读」和训练数据**根本不是一个
分布**。差在三个维度：

1. **说话风格**：训练是「一句一句、刻意、慢」的孤立短语；朗读是**连读**——有连音、弱化、语速更
   快。LoRA 学到的是孤立慢速发音的声学模式，连读时对不上。
2. **词汇 / 内容**：训练是课堂短语，课文是文学语句。LoRA 学的是这个人**特定那些词**的发音；没
   见过的词没覆盖，而且 LoRA 会把输出往「见过的词表」上拽（于是蹦出「老师 / 晚河 / 五百」）。
3. **数据格式**：每条训练样本 = 1 短句 + 静音补到 30s，模型被特化成「短促一下然后大片静音」。
   连续语音的 30s 窗口对它是格式外。

**VAD 切句只缓解了第 3 点（把长音频切回短片段），碰不到第 1、2 点**——那两条是训练时刻进模型里
的。所以切句没能救回准确率。

> 一句话结论：**要识别课文朗读，训练数据就必须来自「课文朗读」这个分布。没有任何推理技巧能补
> 上训练分布的缺口。** 这也和之前「瓶颈在数据不在模型」的结论一致，现在把它落到具体动作上。

---

## 二、策略：把训练分布对齐到「朗读」

三条杠杆，按收益从大到小：

- **① 补录朗读数据（根本解）** —— 让小崔读课文，按句采集成 (音频, 文本) 对。风格=连读、词汇=
  文学，第 1、2 点同时补上。
- **② 掺「长片段」训练（格式对齐）** —— 把同一篇里连续几句**拼成 10–25s 的多句片段**当训练样
  本，让模型学会「连续解码 / 多句一个窗口」，而不只是「短句+静音」。补第 3 点。
- **③ 留一篇完整课文做测试（诚实评估）** —— 整篇 hold out、绝不参与训练，CER 测「没见过的课
  文」上的表现，这才是目标场景的真实指标（顺带补上 roadmap 里一直缺的「目标场景测试集」）。

### 为什么是「按句录」而不是「整段录 + 强制对齐」

整段连续录 + forced alignment 切句，理论上风格最真，但：
- 需要中文强制对齐器（MFA / wav2vec-CTC），对**非典型语音**容易对歪、且是新依赖；
- 小崔可能**读错 / 漏读 / 重读**，对齐出的标签会带噪声——小数据上标签噪声最致命。

所以**第一版用「按句录」**：给一句、读一句、录一句，标签天然干净、复用现有采集页、零新依赖。
连读风格的「跨句协同发音」这一点丢了，但最大的两块收益（词汇 + 句内朗读语速 + 通过拼接得到的
长格式）都拿到了。**若按句录还不够，再上「整段录 + 强制对齐」作为第二版升级。**

---

## 三、采集规格（给小崔录什么、怎么录）

**选材**：挑目标场景那类**文学/课文短句**，覆盖面尽量广（不同课文 = 不同词汇 + 可整篇 hold out）。
- 建议第一批 **5–8 篇**课文，每篇拆成句子，合计 **150–250 句**（和现有 300 短句量级相当才能扭分布）。
- 句子粒度：以**标点（。！？；）断句**，一句一条。太长的句子（>15 字 / 预计 >12s）可在逗号处再断。

**录制方式**（复用现有采集页）：
1. 把每篇课文按句拆好，整理成**待读清单**（见下方 `passages.json`），上传为采集任务。
2. 小崔**按自然朗读语速**读每一句（不要像背单词那样一字一顿，也不要刻意慢——就当在读课文）。
3. 录音规格沿用现有：**16kHz 单声道 wav**，安静环境。
4. **按篇、按句顺序**录（同一篇的句子连续录），保持顺序——拼接长片段和整篇 hold out 都靠它。

**清单格式** `passages.json`（一篇一组、句子有序；既是「读什么」也是后续构建的「分组依据」）：
```json
[
  {"passage": "猫", "sentences": [
    "猫的性格实在有些古怪。",
    "说它老实吧，它的确有时候很乖。",
    "它会找个暖和地方，成天睡大觉，无忧无虑。"
  ]},
  {"passage": "白鹅", "sentences": ["好一个高傲的动物。", "..."]}
]
```
> 用 `scripts/build_passage_dataset.py --emit-prompts prompts.txt` 可把清单里所有句子按序导出成
> 一个 txt，直接在采集页「上传待读文本」即可建任务。

## 四、把录音构建成数据集

录完后用 `scripts/build_passage_dataset.py` 一步生成训练/测试集（**非破坏性**，写到
`dataset/<user>/passage_v1/`，不动现有 train/test）：

```bash
conda activate whisper
python scripts/build_passage_dataset.py \
    --username 小崔 \
    --manifest passages.json \
    --holdout 猫 \                       # 整篇 hold out 做测试（可逗号分隔多篇）
    --existing dataset/小崔/all.json \     # 现有 300 短句并入训练（同人、同分布，留着有益）
    --synth-max-s 25 --synth-gap-s 0.3    # 长片段拼接：连续句拼到 ≤25s、句间留 0.3s 静音
```

它会：
- 按 `text_content` 把清单里每句**匹配到已录的 task 音频**（没录到的会列出来提醒你补录）；
- `--holdout` 指定的整篇 → **test.json**（按句，给 `eval_model.py` 量化 CER）；
- 其余篇的句子 + **现有 300 短句** → **train.json**；
- 对训练篇做**长片段合成**：同篇连续句拼成 ≤25s 的多句 wav（写到 `uploads/<user>/synth/`），
  标签=各句文本拼接（**带标点**，顺带教模型断句/标点）→ 也并入 train.json；
- 打印汇总：匹配/缺失句数、合成片段数、train/test 规模。

## 五、训练（finetune.py 基本不用改，只换数据）

```bash
python finetune.py \
    --base_model whisper-base-models/whisper-small \   # 沿用已采纳的 small（见 finetune.md #4）
    --train_data dataset/小崔/passage_v1/train.json \
    --test_data  dataset/小崔/passage_v1/test.json \
    --output_dir output/小崔/whisper-small-passage-v1 \
    --num_train_epochs 8 --learning_rate 2e-4 \
    --augment_config_path configs/augment.json
```
- 数据集已含长片段（≤25s），`reader.py` 的 `max_duration=30` 能容纳，无需改。
- 早停/选优按 CER（已接好），`test.json` 是 hold-out 篇 → 选出的模型是「对没见过课文最好的」。
- 平衡（可选）：若想让朗读分布占更大权重，可少放点现有短句或对朗读样本过采样——先按默认跑，看
  hold-out CER 再决定。

## 六、评估（两层）

1. **量化**：`eval_model.py` 在 hold-out 篇的 test.json 上算 CER + 整句准确——这是「没见过的课
   文（文学词汇 + 朗读语速）」上的泛化指标，和训练同口径、可比。
   ```bash
   python scripts/eval_model.py output/小崔/whisper-small-passage-v1/checkpoint-final \
       --data dataset/小崔/passage_v1/test.json
   ```
2. **端到端**：把那段 **2 分钟连续《猫》录音**走线上识别路径（VAD 切句 → 拼全文），肉眼/对照
   reference 看是否「能读懂语义」。这是真实目标场景的验收。

> 注意：hold-out 篇是**按句录**的，量化 CER 偏乐观于「真·连续朗读」（少了跨句协同发音）；连续录
> 音那一关才是终判。两者都看。

---

## 七、预期与迭代路线

- **别期待一步到位**：非典型语音 + 文学长句本就难。目标是从「乱码、读不懂」先到「能读懂大意、
  少量错字」，再逐批补数据往下压 CER。
- **迭代闭环**：补一批课文 → 构建 → 训练 → 看 hold-out CER 与误差画像 → 针对高频错字/错词**定向
  补录** → 再训。和短句优化一样，**数据是唯一杠杆**。
- **若按句录到瓶颈**：升级到「整段连续录 + 强制对齐切句」拿回跨句连读分布；或对**已知文本**场景
  改走「参考文本对齐 + 逐字置信度」（但当前是开放转写，暂不走）。
- **产品形态兜底**：若长段始终不稳，把朗读 UX 改成**逐句朗读**（一句一提示一识别），同时贴合训练
  分布和孩子能力，体验更稳。
